# Third Place Japan（TPJ）サイト制作 — 仕様ファイル説明書

このフォルダは、TPJのWebサイトを制作するための「設計図・指示書」一式です。
サイトに公開する中身ではなく、制作の判断基準をまとめた内部ドキュメントです。

---

## はじめに（AI・開発者への最初の指示）

サイト制作を始める前に、まず本フォルダ内の **CLAUDE.md** を読んでください。
CLAUDE.md が全体の親ドキュメントで、そこから他のファイルを参照する構成です。

作業の進め方：
1. CLAUDE.md と、参照される各 md ファイルをすべて読む
2. プロジェクトの目的・全体構成・現フェーズ方針を理解する
3. いきなり実装せず、まず理解の要約と着手プランを提示する
4. 承認を得てから、フェーズごとに実装に進む

---

## 構成ファイル（5ファイル体制）

| ファイル | 役割 |
|---|---|
| **CLAUDE.md** | 全体方針・トーン・確定コピー（親ドキュメント。最初に読む） |
| **DEFINITION.md** | サードプレイスの定義（全コンテンツの思想的な根幹） |
| **INFORMATION-ARCHITECTURE.md** | サイト構造・URL設計・AEO実装・技術スタック |
| **AEO-PLAYBOOK.md** | AI引用獲得（AEO）の施策・タグ正本・業種正本 |
| **STORIES-SITE-SPEC.md** | 記事サイト（/stories/）の実装マスター仕様 |

---

## 重要：公開リポジトリに含めないファイル（2ファイル）

本リポジトリはパブリック（公開）のため、機密性の高い以下2ファイルは**含めません**。

**① EDITORIAL-COMMERCE.md（有償コンテンツの見せ方・価格設計・手数料）**
- 価格設計・手数料率・有償戦略など、外部に出すべきでない情報のため除外。
- 現フェーズは有償展開を保留しているため、サイト制作にも不要。

**② BUSINESS-MODEL.md（事業モデル・上場ロードマップ）**
- 事業収益・上場・スコア評価など、機密情報のため除外。

- 上記2ファイルは CLAUDE.md の参照リストからも削除済みです（5ファイル体制）。
- **VSコード／公開リポジトリのプロジェクトには含めないでください。**
- いずれも将来の有償・事業フェーズで再利用するため、破棄せず
  **非公開の場所（ローカル等）で別途保管**します。

---

## 現フェーズの方針（最重要）

現在は「記事サイト（/stories/）の無償記事を充実させ、アクセスと認知を獲得する」段階です。
以下を必ず守ってください。

- **メインサイト**：思想・基準・世界観を静かに伝える場に徹する。
  CTA・申請導線・お問い合わせ誘導は設置しない（保留）。
- **有償展開**：掲載枠・有償記事・認証マネタイズ・コンサルは一旦保留・停止。
  受付・告知・価格表示は行わない。
- **注力対象**：記事サイトの無償記事（特に神社・山・文化施設などの集客記事）。
- EDITORIAL-COMMERCE.md（有償の見せ方・価格）は将来の再開時に参照する資料であり、
  現フェーズでは実装しない。

---

## 技術スタック（INFORMATION-ARCHITECTURE.md より）

- **メインサイト（法人向け）**：静的1ページ（素のHTML/CSS/JS）
- **記事サイト（個人向け・/stories/）**：11ty（Eleventy / 静的サイトジェネレータ）
- **店舗・記事データ**：JSON等で管理し、テンプレートからページを自動生成（データ駆動）
- **多言語**：/en/ ディレクトリ、hreflang 対応（英語版は後日フェーズ）

---

## 取り扱いについて（パブリックリポジトリ前提）

本リポジトリはパブリック（公開）で運用します。
そのため、機密性の高い事業情報は最初から含めない方針とします。

- **EDITORIAL-COMMERCE.md（価格・有償戦略）と BUSINESS-MODEL.md（事業・上場）は
  本リポジトリに含めません。** 非公開の場所（ローカル等）で別途保管します。
- 本フォルダの md ファイルは「サイト制作の設計図」であり、
  サイトに公開する中身ではありません。可能であれば、これらの md を
  **公開ビルド（サイトの出力先）には含めない**設定にしてください
  （リポジトリ上には存在しますが、サイト訪問者には表示させない運用）。

---

## 推奨する制作ステップ

1. **メインサイト（静的1ページ）** — 構成が固定的で完結しやすい。CLAUDE.md セクション3・6を使用
2. **記事サイト（/stories/）の11ty基盤** — STORIES-SITE-SPEC.md を正本に、
   ディレクトリ・データスキーマ・記事テンプレート・構造化データを設計
3. **最初の「面」を1つ作る** — 例：渋谷×カフェ、または集客記事（神社・山）から着手し、
   AEO-PLAYBOOK.md の「面の構造」に沿って雛形を確立
4. 雛形をもとに記事を量産（STORIES-SITE-SPEC.md の投稿フェーズ計画・KPIを参照）
